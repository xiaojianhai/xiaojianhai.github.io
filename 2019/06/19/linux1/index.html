tar命令

用来解压，打包文件的命令，不能进行压缩操作

命令格式

tar[必要参数][选择参数][文件]

命令功能

用来压缩和解压文件，tar本身不具有压缩功能，通过调用压缩功能实现。

必要参数

-A 新增压缩文件到已存在压缩

-B 设置区块大小

-c 建立新的压缩文件

-x 从压缩文件中提取文件

-t 显示压缩文件的内容

-z 支持gzip解压文件

-j 支持bzip2解压文件

-v 显示操作过程

-k 保留原有文件不覆盖

-W 确认压缩文件正确性

可选参数

-b 设置区块数目

-C 切换到指定目录

-f 指定压缩文件

实例

解压到特定目录

tar -xzvf .gzip -C target_

将/etc目录下的文件全部打包成为/tmp/etc.tar

tar -cvf /tmp/etc.tar /etc

tar -zcvf /tm[/etc.tar.gz /etc

备份所有文件并且保存权限

tar -zcvpf /tmp/etc.tar.gz /etc

chgrp,chown,chmod

chgrp

用来变更文件和目录的群组

chgrp [选项] [组] [文件或目录]

命令参数

-c 当发生改变是输出调试信息

-f 不显示错误信息

-R 递归处理

-v 运行时显示详细的处理信息

chgrp -R bin test6

将test6及其子目录下所有文件和目录群组改为bin

chown

将指定文件的拥有者改为指定的用户或组，用户可以使用户名或者ID

chown [选项] [所有者：组] 文件

命令参数：

-c 显示更改的部分的信息

-f force忽略错误的信息

-h

-R 递归处理

-v 显示处理信息

双短线：

–help 显示帮助信息

–version 显示版本信息

实例：

将log123的拥有者改为mail,群组改为mail

chown mail:mail log123

chmod

改变文件或目录的权限，当用ls -l 显示文件的详细信息，最左边一列为文件的访问权限

形式如下:-rw-r-r-共10位

第一位指定了文件类型，-为文件，d为目录

2-4, 拥有者的权限

5-7, 组的权限

8-10，为其他的权限

命令格式：

chomd [选项] mode file

命令参数：

-f 错误信息不输出

-R 递归处理

-v 显示详细消息

对象：

u；文件的拥有者

g：文件的群组

o: 其他用户群组

a: 所有的用户和群组

权限：
r读：4
w写：2
x执行：1

实例：

将所有用户增加可执行权限

chmod -f a+x log2013

增加用户执行权限，增加组和其他的写和执行权限

chmod u+x,go+wx log2013

设置为所有用户读写

chmod 777 log 2013== chmode a+rwx log2012
